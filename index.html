BAFFLE

BAFFLE is a Behavioral Active Fingerprinting tool for 802.11 APs, operating entirely within the Link Layer[1]. It sends non-standard Probe Request and Authentication Request frames and distinguishes between different AP chipset-driver combinations based on their observed responses.

BAFFLE is written in Ruby and uses LORCON and libpcap (primarily with the Atheros madwifi combination).

BAFFLE was presented and demoed at Shmoocon 2008: [presentation slides][2]

BAFFLE was also presented and demoed at Blackhat USA 2008.

Our Toorcon X presentation went deeper in the details of Ruby implementation and our suggested technique for finer AP fingerprinting using timestamp clock skew in beacon frames.

We have created a simple video demostration of how BAFFLE works.

Code available @ github under the GPL v3.

You can also install it via gem install dxoigmn-baffle, assuming you have http://gems.github.com/ in your gem sources (see link for details).

[1]
